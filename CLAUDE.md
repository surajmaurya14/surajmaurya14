## What this repo is

This is a GitHub *profile* repository (`surajmaurya14/surajmaurya14`). The single deliverable is `README.md`, which GitHub renders on the user's profile page. There is no application code, no build, no tests, and no package manager.

## Architecture: README + two scheduled workflows that regenerate dynamic content

Most of the README's "live" widgets are static SVGs generated on a schedule by GitHub Actions and committed back into this repo. Editing those SVGs by hand will be silently overwritten on the next workflow run.

- **`.github/workflows/grs.yml`** ("Update README cards") → daily at 03:00 UTC, plus `workflow_dispatch`. Uses `readme-tools/github-readme-stats-action@v1` (the maintainer-blessed Action route from `anuraghazra/github-readme-stats`) to render two SVGs locally on the runner, then commits them to **`master`** under `profile/`:
  - `profile/stats.svg`
  - `profile/top-langs.svg`
  - The README references these via relative paths (`./profile/...`).
  - This route exists because the public `github-readme-stats.vercel.app` deployment is paused. **Do not re-introduce `github-readme-stats.vercel.app` URLs** unless the upstream service is confirmed back online.

- **`.github/workflows/snake.yml`** ("Generate Snake") → daily at 00:00 UTC, plus `workflow_dispatch`, plus on every push to `master`. Uses `Platane/snk/svg-only@v3` to render snake-eats-contributions SVGs and `crazy-max/ghaction-github-pages@v4` to push them to a separate **`output`** branch:
  - `github-contribution-grid-snake.svg`
  - `github-contribution-grid-snake-dark.svg`
  - The README references these via `raw.githubusercontent.com/.../output/...`.
  - The `output` branch is action-managed; never merge it into `master` and never edit it by hand.

Both workflows authenticate with the auto-injected `secrets.GITHUB_TOKEN`. No PAT is required.

## Other widgets in the README (third-party, fetched at view time)

These render directly from external services and are NOT regenerated by workflows. If one breaks, the fix is usually swapping the host:

- `readme-typing-svg.demolab.com` — typing hero (replaced the dead `*.herokuapp.com` host)
- `komarev.com/ghpvc/` — profile views counter
- `streak-stats.demolab.com` — streak card (replaced the dead `github-readme-streak-stats.herokuapp.com`)
- `github-readme-activity-graph.vercel.app` — activity graph
- `github-profile-trophy.vercel.app` — trophies
- `img.shields.io` — all badges

Two stale services to **never reintroduce**: `github-readme-streak-stats.herokuapp.com` (Heroku free tier shut down Nov 2022) and `profile-summary-for-github.com` (defunct, was wrapping stat images as anchor links in old versions).

## Theme convention

`tokyonight` is the consistent theme across stats, top-langs, streak, and trophies. **Activity graph uses the slug `tokyo-night` (with a dash)** — different project, different naming. Don't normalize one to the other.

## Common operations

- **Manually refresh cards now (instead of waiting for cron):**
  ```
  gh workflow run grs.yml -R surajmaurya14/surajmaurya14
  gh workflow run snake.yml -R surajmaurya14/surajmaurya14
  ```
- **Check recent runs / debug failures:**
  ```
  gh run list -R surajmaurya14/surajmaurya14 --limit 5
  gh run view <run-id> -R surajmaurya14/surajmaurya14 --log-failed
  ```
- **Required GitHub setting** (default-on for repos created since early 2023): *Settings → Actions → General → Workflow permissions → "Read and write"*. Without this, both workflows fail to push their generated SVGs back to the repo.

## When editing the README

- Verify that any new external image URL is from a maintained host (Vercel/demolab/komarev/shields). Heroku-hosted README widgets are dead.
- The README's `<picture>` block for the snake declares both light and dark variants — keep both `<source>` lines in sync if you change the path scheme.
- Tables and centered `<p>`-wrapped images render fine in GitHub's markdown; bare HTML `<div>` does not center reliably.
